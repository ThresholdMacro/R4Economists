Simple interest rate calculations

  • The return from an interest rate over 2 periods. 𝑖 =0.05 and 𝑡 =2 .
    • accum holds the result of the calculation of 1+𝑖*𝑡 for the values of 𝑖 and 𝑡.
    • Exponentiate the quantity (1+𝑖) to the 𝑡 th power.
    • Create a sequence for  𝑡 & recalculate.
    • Create a dataframe.

Interest rate calculation in console

i  <-  0.05 
t <-  2 
accum  <-  1  +  i * t
accum
[1] 1.1
accum2 <- ( 1  +  i ) ^ t
accum2
[1] 1.1025
t <-  seq ( from =  0 , to =  5 , by =  0.5 )
accum3  <- ( 1  +  i ) ^ t 
accum3
 [1] 1.000000 1.024695 1.050000 1.075930 1.102500 1.129726 1.157625 1.186213
 [9] 1.215506 1.245523 1.276282
accum4 <- data.frame ("Period" = t, "Data"=(1+i)^t)

Create a function

  • User-created functions add to inbuilt capabilities.
    • A function is assigned a name (simple), takes parameters ( ir , time), returns a result
    • A function will automatically return the last calculation. It is good practice to define what is returned
    • Let’s try it.

Write a function in console

simple <-  function ( ir , time){
1  +  ir * time
}
simple <-  function ( ir , time){
srs <-( 1  +  ir ) ^ time 
return( srs )
}

6 Basic data types

  1. double or numeric (real or decimal) (2, 15.5)

  2. character (“a”, “swc”, “name”)

  3. integer (2L the L tells R to use this as an integer)

  4. logical (TRUE/FALSE)

  5. complex (complex numbers with real and imaginary parts)

  6. raw (not normally used in day-to-day usage, but may be important in messaging, piping, storage. Example, hex colours)

5 Basic data structures

  1. atomic vector (a collection of data in single row (or column) of one type of data).

  2. list (a vector of objects, can contain multiple types of data).

  3. matrix (a vector with more than two dimensions).

  4. dataframe (a matrix with special/useful characteristics. The de facto data structure for most tabular data ).

  5. factor ( used to represent categorical data. Factors can be ordered or unordered and are an important class for statistical analysis and for plotting). All data structures have dimensions, names, types, contents.
